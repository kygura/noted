A new visualization tool for Noted:

**The grid IS the graph, but with no visible edges.** Proximity, cluster density, and grouping encode the semantic relationships. The engine's output is spatial arrangement of cards, not a node-edge diagram. You read structure by scanning, not by tracing lines.

**What this actually looks like:**

The default view is a responsive card grid. Cards are markdown notes rendered as readable previews — not truncated titles, not icons. Enough content to recognize the thought at a glance.

Cards don't sit in folders. They sit in **named semantic clusters** — visually grouped by proximity or subtle visual treatment (background tint, border, label). Cluster names are engine-generated, not user-assigned. No explicit edges drawn between anything.

**Navigation is query-driven, not spatial.** You type a concept — not a search term, a concept — and the grid reorganizes. Cards semantically close to the query surface and group. Cards far from it recede or disappear. The layout is the answer.

**Keyboard-first throughout:**
- `n` → Write surface
- `/` → query input, grid responds live
- Arrow keys → move between cards
- `Enter` → open note in Write surface
- `Esc` → back to grid
- `Tab` → cycle between clusters

No mouse required for core navigation. The grid feels like a terminal that renders beautifully.

**Where this breaks from hierarchical thinking:**

Folders answer the question *where did I put this*. The grid answers *what is this near*. You never file a note. You never navigate a tree. You query a concept and the structure reveals itself around that concept. Every query produces a different valid organization of the same corpus. There is no canonical arrangement — only contextually relevant ones.

**The Write surface stays unchanged.** Full-screen, blank, no chrome. The grid is what you return to.
